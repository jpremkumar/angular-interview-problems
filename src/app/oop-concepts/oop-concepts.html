Core OOP Pillars Encapsulation, Abstraction, Inheritance, Polymorphism

What are the four main pillars of Object-Oriented Programming? Briefly describe each.

  • Encapsulation: Bundling data (attributes) and methods that operate on the data within a single unit (class). Restricts direct access to some of an object's components.
  • Abstraction: Hiding complex implementation details and showing only essential features of the object.
  • Inheritance: Mechanism where a new class (subclass/derived class) inherits properties and methods from an existing class (superclass/base class).
  • Polymorphism: Ability of an object to take on many forms. Commonly, "one interface, multiple functions" (e.g., method overriding).

 SOLID Principles

Can you name and briefly explain the SOLID principles?

  • S - Single Responsibility Principle (SRP): A class should have only one reason to change.
  • O - Open/Closed Principle (OCP): Software entities (classes, modules, functions) should be open for extension but closed for modification.
  • L - Liskov Substitution Principle (LSP): Subtypes must be substitutable for their base types.
  • I - Interface Segregation Principle (ISP): Clients should not be forced to depend on interfaces they do not use.
  • D - Dependency Inversion Principle (DIP): High-level modules should not depend on low-level modules. Both should depend on abstractions. Abstractions should not depend on details. Details should depend on abstractions.

(Code examples for SOLID are extensive and best explored in dedicated resources.)

 Coupling and Cohesion

What do coupling and cohesion mean? Is high or low coupling/cohesion generally preferred, and why?

  • Coupling: The degree of interdependence between software modules. Low coupling is preferred.
  • Cohesion: The degree to which the elements inside a module belong together. High cohesion is preferred.

 OOP in JavaScript/TypeScript

How does JavaScript (ES6+) support OOP concepts? How does TypeScript enhance this?

JS (ES6+) provides classes, inheritance (`extends`), constructors. TypeScript adds static typing, interfaces, access modifiers (`public`, `private`, `protected`), abstract classes, generics, enums, etc., making OOP more robust.
