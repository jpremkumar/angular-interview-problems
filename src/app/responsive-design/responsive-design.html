Responsive Design Techniques

This section demonstrates basic responsive design. The layout of the cards below will change based on the screen width.

  • Large screens (>= 960px): Cards are displayed in a row (3 per row).
  • Medium screens (>= 600px and < 960px): Cards are displayed in a row (2 per row).
  • Small screens (< 600px): Cards are stacked vertically (1 per row).

Resize your browser window to see the changes.

Problem Statement / Interview Scenario:

An application page displays a series of product information cards. On desktop, these cards should appear in a grid-like fashion (e.g., 3 or 4 cards per row). On tablets, fewer cards should appear per row (e.g., 2 per row). On mobile phones, the cards should stack vertically, each taking the full width available. The content within the cards (text, images) should also adjust gracefully.

  1. What are the primary CSS techniques you would use to achieve this responsive layout (e.g., Flexbox, Grid, Media Queries)? Compare and contrast CSS Grid and Flexbox for this specific card layout scenario. When might you choose one over the other, or use them together?
  2. How would you define breakpoints for different screen sizes? What factors influence this decision? What does "mobile-first" development mean in this context, and what are its advantages?
  3. Describe how you would ensure images within the cards are responsive (e.g., using max-width: 100%;, <picture> element, srcset attribute).
  4. If you needed to change component behavior or load different content based on screen size (not just styling), what Angular-specific tools could you use? Explain how the BreakpointObserver service from the Angular CDK LayoutModule works and how you might use it.
  5. What are some common pitfalls or challenges when implementing responsive designs, such as dealing with navigation, complex tables, or large forms on small screens?
  6. How would you test the responsiveness of your application across different devices and screen sizes (e.g., browser dev tools, emulators, real devices)?
  7. Explain the difference between "adaptive" and "responsive" design. Which approach is primarily demonstrated in this example?
  8. How can CSS Custom Properties (variables) be utilized to make responsive designs more maintainable, especially for themeable values (e.g., font sizes, spacing) that might change across breakpoints?
